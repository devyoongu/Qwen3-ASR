## 3. 오류 패턴 분석

### 3-1. 전화번호·코드 오인식 (4건)

TTS가 숫자 코드(`1544-4500`, `1588-5000`, `2024-09-0031`, `A-1-2-3-4-5`)를 각 자리 발음으로 읽어 STT가 의미 없는 한글 조합으로 전사. 전화번호 특화 후처리 없이는 해결이 어려운 구조적 한계.

### 3-2. 기술 용어 오인식 (2건)

- `stream_player_put_frame` → "스트림 플레이어 피트 플레이어" — 영문 코드 심볼을 한글 발음으로 잘못 해석
- `pending_data` → "펜딩 데이터" — 음독화. 문맥상 이해 가능한 수준

### 3-3. 브랜드명 오인식 (2건)

- "나비엔" ↔ "나비앤" (007): 단모음/이중모음 혼동
- "경동나비엔은" → "경동나비에는" (092): 브랜드명+조사 경계 오인식

### 3-4. 단어 단위 오인식 (5건)

짧고 발음이 유사한 단어에서 발생:
- "더" → "너" (#073)
- "도서 산간" → "노서 상간" (#090): ㄷ/ㄴ, 산/상 혼동
- "붙여두세요" → "쳐두세요" (#118): 앞부분 누락
- "끌" → "을" (#128): 청각 유사음 혼동

### 3-5. 이중 공백 삽입 (7건)

스트리밍 청크 경계에서 텍스트가 이어붙여질 때 일부 구간에 이중 공백 삽입. 후처리에서 `strip()` / 정규화로 제거 가능.
